### Added
- When navigating to a segment, WEBKNOSSOS now jumps to the stored anchor position instantly and validates it against the actual voxel data afterwards. If the anchor turns out to be stale (e.g. it was overwritten), the segment is searched for nearby (using the segment index if available, otherwise scanning the surrounding buckets across magnifications) and the anchor is updated. The validation is aborted automatically if the segment is removed or re-anchored (e.g. by brushing) in the meantime. If the segment cannot be located, a warning is shown with the option to remove it. This works with mapped ids (merger mode, JSON/HDF5 mappings, proofreading) and transformed datasets.
